# LP AI Builder 画面設計書（制作ツール寄り）

## 1. 文書概要

### 1-1. 目的

本書は、LP AI Builder の画面構成、各画面の役割、表示項目、操作要素、遷移、レイアウト方針を定義することを目的とする。

本システムは**制作ツール寄り**のWebアプリとして設計するため、単純な一発生成ではなく、以下を重視する。

- 情報入力のしやすさ
- セクション単位の編集性
- リアルタイムプレビュー
- 再編集しやすい操作性
- コード生成までの導線の明確化

## 4. 全体UI方針

### 4-1. 基本レイアウト方針

制作ツール寄りのため、**編集画面を中心とした2カラムまたは3カラム構成**とする。

#### 推奨構成

- 左: ナビゲーション / セクション一覧
- 中央: 入力フォーム
- 右: リアルタイムプレビュー

### 4-2. UIの基本思想

- すぐ編集できる
- どこを変えたか分かる
- 変更結果が即見える
- AI生成に頼りすぎず、人が最後に仕上げやすい

### 4-3. レスポンシブ方針

- PCを主対象に設計
- タブレットで最低限操作可能
- スマホは閲覧寄り
- SPプレビューは編集画面内で切替表示

## 8. 主要操作シナリオ

### 8-1. 新規LPを作る場合

1. ダッシュボードで新規作成を押す
2. 新規LP作成画面で基本情報を入力
3. 編集画面に遷移
4. デザインやセクションを調整
5. AIで文言を生成
6. プレビュー確認
7. コード出力
8. 保存またはダウンロード

### 8-2. 既存LPを修正する場合

1. プロジェクト一覧を開く
2. 対象プロジェクトを選択
3. 編集画面で修正
4. 必要ならAIで差し替え
5. 再度コード出力

## 9. UI上の重要設計ポイント

### 9-1. 編集の迷子を防ぐ

- 左ナビで現在位置を明確化
- 選択中セクションをハイライト
- プレビュー側も該当セクションへスクロール連動

### 9-2. 入力負荷を下げる

- 初期値を多めに持つ
- セクションテンプレートを準備
- AI生成は補助として使う

### 9-3. 制作ツールらしさを出す

- 細かいレイアウト指定
- セクションごとの個別設定
- 並び替えの自由度
- コード出力時の粒度指定

## 12. 結論

本アプリの画面設計は、**編集しやすさ・見ながら作れること・再利用しやすさ**を重視して構成する。

特に中核となるのは SCR-004 のLP編集画面であり、ここで

を一連で完結できる設計にする。
